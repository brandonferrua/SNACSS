File Structure

SNACSS Core

SNACSS is built with an Object Oriented Programatic approach in mind. The File structure consists of partials broken out into their own granular @mixin that can be turned on and off based on your needs and accectable file size. The advanced docs will provide more insight into those components. Place the core package within your project, in my project I placed with /assets/snacss/. The package is now available to your project and in the next section, I will go over the basics of initializing the package.

Initialization

Now that you have SNACSS available to your project, a few includes will have you up and running in seconds. Within the folder you dropped the SNACSS core, so my example would be /assets/, creating your css folder. This is where all your project level styles will be held. For SNACSS to init, you only need two files but I recommend you keep the OOP approach intact by also creating a /theme/ where you will place all your overrides or styles that may not be available within the SNACSS framework. To begin, create two files within /assets/css/, you will need a core.scss and _vars.scss. I'm assuming if you're looking for a SASS framework you will know how to compile a sass file but if you don't, do the following steps:

  1. open terminal
  2. dig down to your project, for me its ~/username/Applications/MAMP/htdocs/SNACSS/
  3. Run this cmd, sass --watch assets

Your project should now be spitting out a core.css file now.

Now lets setup your core.scss file to initialize SNACSS. Below is a basic example of initialization:

// Local Project Overrides
@import "vars";

// Import SNACSS Framework
@import "../snacss/settings";
@import "../snacss/init";

// Local Project Styles
@import "theme/core";
